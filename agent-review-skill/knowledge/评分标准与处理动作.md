# 评分标准与处理动作

> **用途**：定义Agent体系审查的评分等级和处理动作
> **版本**：v1.0 | 2026-06-17
> **适用范围**：四类审查子Agent统一使用

---

## 一、评分等级定义

```
┌──────────────────────────────────────────────────────────────┐
│                    评分等级定义                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【A级】>= 90分                                               │
│  ├─ 状态：pass                                               │
│  ├─ 含义：完全合规，无需修改                                 │
│  └─ 处理：继续审查下一维度                                   │
│                                                              │
│  【B级】75-89分                                               │
│  ├─ 状态：pass                                               │
│  ├─ 含义：基本合规，有改进建议                               │
│  └─ 处理：继续审查，输出改进建议                             │
│                                                              │
│  【C级】60-74分                                               │
│  ├─ 状态：warning                                            │
│  ├─ 含义：部分不合规，需修改                                 │
│  └─ 处理：记录问题，继续审查，出具改进建议                   │
│                                                              │
│  【D级】40-59分                                               │
│  ├─ 状态：error                                              │
│  ├─ 含义：严重不合规，需重写                                 │
│  └─ 处理：阻断，需修正后重新审查对应维度                     │
│                                                              │
│  【F级】< 40分                                                │
│  ├─ 状态：error                                              │
│  ├─ 含义：完全不合格，退回重写                               │
│  └─ 处理：阻断，需完全重写后重新审查                         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 二、维度评分计算

### 2.1 检查项评分

每个检查项采用二元评分：

| 检查结果 | 得分 |
|----------|------|
| pass | 100分 |
| warning | 60分 |
| error | 0分 |

### 2.2 维度加权分

```
维度得分 = Σ(检查项得分) / 检查项数量
维度加权分 = 维度得分 × 权重
```

### 2.3 综合评分

```
综合评分 = Σ(各维度加权分)
```

## 四、处理动作映射

### 4.1 单维度处理

| 等级 | 状态 | 处理动作 |
|------|------|----------|
| A级 | pass | ✅ 继续审查下一维度 |
| B级 | pass | ✅ 继续审查，输出改进建议 |
| C级 | warning | ⚠️ 记录问题，继续审查 |
| D级 | error | ❌ 阻断，需修正后重新审查 |
| F级 | error | ❌ 阻断，需完全重写 |

### 4.2 整体处理

| 条件 | 处理 |
|------|------|
| 任意维度为D级或F级 | 整体require_revision=true |
| 两个及以上维度为C级 | 整体require_revision=true |
| 综合评分<60分 | 整体require_revision=true |
| 其他情况 | 整体require_revision=false |

## 五、问题严重程度定义

### 5.1 三级严重程度

```
┌──────────────────────────────────────────────────────────────┐
│                    问题严重程度                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【P0 - 高严重性】                                            │
│  ├─ 触发条件：检查项结果为error                              │
│  ├─ 处理：必须修正                                           │
│  └─ 阻断：是                                                 │
│                                                              │
│  【P1 - 中严重性】                                            │
│  ├─ 触发条件：检查项结果为warning                            │
│  ├─ 处理：建议修正                                           │
│  └─ 阻断：否                                                 │
│                                                              │
│  【P2 - 低严重性】                                            │
│  ├─ 触发条件：存在改进建议但检查通过                         │
│  ├─ 处理：可选优化                                           │
│  └─ 阻断：否                                                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 六、升级规则

### 6.1 自动升级条件

```
┌──────────────────────────────────────────────────────────────┐
│                    升级规则                                    │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【规则一】单个维度出现>=3个error → 该维度自动降为F级         │
│                                                              │
│  【规则二】两个及以上维度为C级 → 整体require_revision=true    │
│                                                              │
│  【规则三】任意维度为D级或F级 → 整体require_revision=true     │
│                                                              │
│  【规则四】连续审查不通过超过2次 → 升级人工介入               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 人工介入条件

- 同一审查点连续2次为D级或F级
- 综合评分连续2次低于60分
- 存在无法自动判断的矛盾

## 七、评分示例

### 示例1：主Agent审查

| 维度 | 权重 | 得分 | 加权分 |
|------|------|------|--------|
| 流程协调 | 30% | 90 | 27 |
| 任务拆解 | 25% | 85 | 21.25 |
| KVP验证 | 25% | 80 | 20 |
| 错误处理 | 20% | 70 | 14 |

**综合评分**：27 + 21.25 + 20 + 14 = 82.25 ≈ **82分（B级）**

**处理**：pass，输出改进建议

---

### 示例2：子Agent审查（有error）

| 维度 | 权重 | 得分 | 加权分 |
|------|------|------|--------|
| 输入契约 | 25% | 70 | 17.5 |
| 输出契约 | 25% | 60 | 15 |
| 职责边界 | 30% | 40 | 12 |
| 知识调用 | 20% | 80 | 16 |

**综合评分**：17.5 + 15 + 12 + 16 = 60.5 ≈ **60分（C级）**

**处理**：warning，职责边界维度存在error，需修正
