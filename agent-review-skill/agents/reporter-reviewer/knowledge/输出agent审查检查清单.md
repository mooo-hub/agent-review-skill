# 输出Agent审查检查清单

> **用途**：审查报告生成Agent是否符合Reporter角色规范
> **版本**：v1.0 | 2026-06-17
> **来源**：[[../knowledge/四角色架构规范]]

---

## 核心审查原则

```
┌──────────────────────────────────────────────────────────────┐
│                    审查核心原则                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【原则一：模板驱动】                                          │
│  按固定模板生成报告，输出格式确定，不依赖上下文判断             │
│                                                              │
│  【原则二：数据来源标注】                                      │
│  每个章节标注数据来自哪个Agent，确保可追溯                     │
│                                                              │
│  【原则三：先数据后解读】                                      │
│  先展示数据表格，再进行文字解读                                │
│                                                              │
│  【原则四：降级处理】                                          │
│  数据缺失时在报告对应位置注明"数据暂缺"，不中断生成             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度一：格式规范（权重35%）

```
┌──────────────────────────────────────────────────────────────┐
│                    格式规范检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [R-1.1] 存在明确的报告模板（非文字描述，直接内嵌）           │
│    ├─ 检查：是否有报告模板章节                                │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：直接内嵌Markdown模板                          │
│                                                              │
│  □ [R-1.2] 报告模板包含标题、摘要、正文、来源等核心章节         │
│    ├─ 检查：模板是否包含核心章节                              │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：至少包含标题、摘要、正文、来源引用             │
│                                                              │
│  □ [R-1.3] 支持多种输出格式（Markdown/PDF/DOCX/JSON）          │
│    ├─ 检查：是否说明支持多种输出格式                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：在输出契约中列出支持的格式                    │
│                                                              │
│  □ [R-1.4] 数字精度有规范（百分比/金额/时间格式）               │
│    ├─ 检查：是否定义数字格式规范                              │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：说明百分比保留几位小数等                       │
│                                                              │
│  □ [R-1.5] 使用ALWAYS标注格式约束                              │
│    ├─ 检查：是否有明确的格式约束                              │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：使用"ALWAYS使用xxx格式"表述                   │
│                                                              │
│  □ [R-1.6] 触发条件明确（仅当数据聚合完毕后触发）               │
│    ├─ 检查：是否说明何时触发报告生成                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：明确"仅当主Agent确认所有子任务完成后触发"      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度二：完整性（权重35%）

```
┌──────────────────────────────────────────────────────────────┐
│                    完整性检查项                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [R-2.1] 汇总了所有上游Agent的结构化数据                     │
│    ├─ 检查：是否说明汇总哪些Agent的输出                       │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：列出上游Agent及其输出字段                     │

│  □ [R-2.2] 每个章节标注了数据来源Agent                         │
│    ├─ 检查：是否要求标注数据来源                              │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：使用"来源：xxx Agent"标注                     │
│                                                              │
│  □ [R-2.3] 有完整性检查清单                                    │
│    ├─ 检查：是否有报告完整性检查清单                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：列出必须包含的章节和字段                      │
│                                                              │
│  □ [R-2.4] 数据缺失时在报告对应位置注明"数据暂缺"              │
│    ├─ 检查：是否定义数据缺失时的处理                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：明确标注缺失字段                              │
│                                                              │
│  □ [R-2.5] 不因部分数据缺失而中断生成                          │
│    ├─ 检查：是否说明部分缺失不影响整体生成                    │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：继续生成，在缺失位置标注                      │
│                                                              │
│  □ [R-2.6] 输出包含metadata（生成时间、版本、分析类型）         │
│    ├─ 检查：是否要求输出元数据                                │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：在报告头部包含metadata                        │
│                                                              │
```

```
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：先摘要，后详细章节                            │
│                                                              │
│  □ [R-3.3] 遵循"先核心指标，后辅助参考"原则                    │
│    ├─ 检查：是否要求核心指标优先                              │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：核心指标放在前面                              │

│    ├─ 检查：是否定义表述风格                                  │
```

## 常见问题与正确做法

| 问题 | 错误做法 | 正确做法 |
|------|----------|----------|
| 无报告模板 | 文字描述报告结构 | 直接内嵌Markdown模板 |
| 无数据来源 | 报告中不标注来源 | 每个章节标注来源Agent |
| 数据缺失中断 | 部分数据缺失时报错 | 标注"数据暂缺"并继续 |
| 先解读后数据 | 先写文字解读 | 先展示数据表格 |

## 评分标准

| 检查项数量 | 评分计算 |
|-----------|---------|
| pass数量 | 100分/项 |
| warning数量 | 60分/项 |
| error数量 | 0分/项 |

```
维度得分 = Σ(检查项得分) / 检查项数量
维度加权分 = 维度得分 × 权重
综合评分 = Σ(各维度加权分)
```
