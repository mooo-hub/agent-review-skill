# 审核Agent审查检查清单

> **用途**：审查审核Agent是否符合Reviewer角色规范
> **版本**：v1.0 | 2026-06-17
> **来源**：[[../knowledge/四角色架构规范]]

---

## 核心审查原则

```
┌──────────────────────────────────────────────────────────────┐
│                    审查核心原则                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【原则一：审查独立】                                          │
│  审核Agent不参与生产流程，只在生产完成后介入，保证审查客观性    │
│                                                              │
│  【原则二：意见出具】                                          │
│  只出具审查意见，不自行修改内容，不做决策                      │
│                                                              │
│  【原则三：结构化输出】                                        │
│  审查输出为结构化JSON，定义pass/warning/error三状态            │
│                                                              │
│  【原则四：升级机制】                                          │
│  score<60或存在error问题时，返回require_revision=true          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度一：独立性（权重35%）

```
┌──────────────────────────────────────────────────────────────┐
│                    独立性检查项                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [V-1.1] 不参与生产流程（只在生产完成后介入）                │
│    ├─ 检查：是否明确不参与生产流程                            │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：明确"不参与生产流程，只在生产完成后介入"       │
│                                                              │
│  □ [V-1.2] 不自行修改被审查内容                                │
│    ├─ 检查：是否明确不修改内容                                │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：明确"只出具意见，不自行修改"                  │
│                                                              │
│  □ [V-1.3] 只出具意见，由主Agent决策是否返工                   │
│    ├─ 检查：是否明确不做决策                                  │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：返回require_revision，由主Agent决策           │
│                                                              │
│  □ [V-1.4] 不在审查结果中直接给出修改后文字                    │
│    ├─ 检查：是否避免直接给出修改后文字                        │
│    ├─ 错误级别：warning                                        │

│                                                              │
│  □ [V-1.5] 审核节点触发时机明确（在哪个Agent之后执行）          │
│    ├─ 检查：是否说明审核节点触发时机                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：明确"在xxx Agent完成后执行"                   │
│                                                              │
│  □ [V-1.6] 不同审核节点加载对应规则集（禁止混用）               │
│    ├─ 检查：是否禁止不同节点混用规则集                        │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：明确"禁止同时引用两个节点的检查清单"          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 常见问题与正确做法

| 问题 | 错误做法 | 正确做法 |
|------|----------|----------|
| 参与生产流程 | 审核Agent也执行分析 | 明确只在生产完成后介入 |
| 直接修改内容 | 在审查结果中给出修改后文字 | 只给出建议，不直接修改 |
| 无升级条件 | 审核失败无明确处理 | 定义require_revision和升级规则 |
| 混用规则集 | 同一审核使用多个检查清单 | 每个节点加载对应规则集 |

## 评分标准

| 检查项数量 | 评分计算 |
|-----------|---------|
| pass数量 | 100分/项 |
| warning数量 | 60分/项 |
| error数量 | 0分/项 |

```
维度得分 = Σ(检查项得分) / 检查项数量
维度加权分 = 维度得分 × 权重
综合评分 = Σ(各维度加权分)
```
