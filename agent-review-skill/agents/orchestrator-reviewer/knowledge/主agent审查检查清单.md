# 主Agent审查检查清单

> **用途**：审查主调度Agent是否符合Orchestrator角色规范
> **版本**：v1.0 | 2026-06-17
> **来源**：[[../knowledge/四角色架构规范]]

---

## 核心审查原则

```
┌──────────────────────────────────────────────────────────────┐
│                    审查核心原则                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【原则一：主Agent集权】                                       │
│  所有决策权（重试、终止、路由）归主Agent，子Agent不做策略判断    │
│                                                              │
│  【原则二：唯一入口】                                          │
│  主Agent是唯一面向用户的入口，不直接执行业务逻辑               │
│                                                              │
│  【原则三：KVP强制验证】                                       │
│  必须验证子Agent输出，禁止在未验证时标记"called":true          │
│                                                              │
│  【原则四：失败可观测】                                        │
│  每次失败都必须有结构化日志                                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度一：流程协调（权重30%）

```
┌──────────────────────────────────────────────────────────────┐
│                    流程协调检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [O-1.1] 定义了完整的调度流程图                              │
│    ├─ 检查：SKILL.md中是否有流程图/流程描述                    │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：使用ASCII图或步骤列表展示完整流程              │
│                                                              │
│  □ [O-1.2] 识别串行/并行节点并标注                             │
│    ├─ 检查：是否明确标注哪些步骤串行、哪些并行                 │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：在流程图或描述中标注"(并行)"或"(串行)"         │
│                                                              │
│  □ [O-1.3] 维护全局上下文数据结构                              │
│    ├─ 检查：是否定义了session上下文结构                        │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：定义JSON格式的上下文数据结构                   │
│                                                              │
│  □ [O-1.4] 唯一面向用户的入口，不直接执行业务逻辑               │
│    ├─ 检查：职责边界是否明确"不直接执行业务逻辑"               │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：在"不负责"部分明确列出                         │
│                                                              │
│  □ [O-1.5] 不直接生成最终报告                                  │
│    ├─ 检查：职责边界是否明确"不生成最终报告"                   │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：报告生成委托给reporter Agent                   │
│                                                              │
│  □ [O-1.6] 不做质量审查（委托审查Agent）                        │
│    ├─ 检查：职责边界是否明确"不做质量审查"                     │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：审查委托给audit-agent                          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度二：任务拆解（权重25%）

```
┌──────────────────────────────────────────────────────────────┐
│                    任务拆解检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [O-2.1] 子Agent注册表存在且完整                             │
│    ├─ 检查：是否有"子Agent列表与职责"章节                      │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：表格列出所有子Agent及职责                      │
│                                                              │
│  □ [O-2.2] 每个子任务有明确的task_type                         │
│    ├─ 检查：调用格式是否包含task_type字段                      │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：collect|analyze|transform等类型               │
│                                                              │
│  □ [O-2.3] 每个子任务定义了输入契约（input schema）             │
│    ├─ 检查：调用格式是否包含input字段                          │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：明确传入哪些数据给子Agent                      │
│                                                              │
│  □ [O-2.4] 每个子任务定义了输出契约（output schema）            │
│    ├─ 检查：调用格式是否说明输出字段                           │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：明确子Agent输出写入哪个上下文字段              │
│                                                              │
│  □ [O-2.5] 子任务之间的依赖关系已描述                          │
│    ├─ 检查：是否说明哪些任务依赖其他任务输出                   │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：流程图或表格描述依赖关系                       │
│                                                              │
│  □ [O-2.6] 调度消息格式包含task_id/task_type/input/timeout     │
│    ├─ 检查：调用JSON是否包含这些必要字段                       │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：使用标准调度消息格式                           │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度三：KVP验证（权重25%）

```
┌──────────────────────────────────────────────────────────────┐
│                    KVP验证检查项                               │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [O-3.1] 每个子Agent调用后有KVP验证点                        │
│    ├─ 检查：是否有KVP列表，每个子Agent对应一个验证点           │

│    ├─ 错误级别：error                                         │
│    └─ 正确做法：验证格式包含output_sample字段                  │
│                                                              │
│  □ [O-3.3] 验证失败有明确错误输出                              │
│    ├─ 检查：是否定义验证失败时的输出格式                       │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：定义error字段和错误描述                        │
│                                                              │
│  □ [O-3.4] 禁止在未验证时输出"called":true                     │
│    ├─ 检查：是否明确禁止此行为                                 │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：在规则中明确禁止                               │
│                                                              │
│  □ [O-3.5] KVP列表在SKILL.md中完整列出                         │
│    ├─ 检查：KVP列表是否完整                                   │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：所有子Agent都在KVP列表中                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度四：错误处理（权重20%）

```
┌──────────────────────────────────────────────────────────────┐
│                    错误处理检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │

│    ├─ 检查：是否定义数据缺失时的处理                           │
│    ├─ 错误级别：warning                                        │

│                                                              │
│  □ [O-4.3] 审核未通过有返工流程                                │
│    ├─ 检查：是否定义审核失败时的返工机制                       │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：定义返工条件和返工步骤                         │
│                                                              │

│    ├─ 检查：是否定义升级人工介入条件                           │
│    ├─ 错误级别：warning                                        │

│  □ [O-4.5] 每次失败有结构化日志                                │
│    ├─ 检查：是否定义失败日志格式                               │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：日志包含msg_id、from、error_code              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 常见问题与正确做法

| 问题 | 错误做法 | 正确做法 |
|------|----------|----------|
| 无KVP验证 | 直接调用子Agent后继续 | 调用后必须验证输出 |
| 职责不清 | 主Agent既协调又分析 | 明确"不负责"边界 |
| 无错误处理 | 子Agent失败后崩溃 | 定义重试和升级机制 |
| 无流程图 | 只有文字描述 | 使用ASCII流程图 |

## 评分标准

| 检查项数量 | 评分计算 |
|-----------|---------|
| pass数量 | 100分/项 |
| warning数量 | 60分/项 |
| error数量 | 0分/项 |

```
维度得分 = Σ(检查项得分) / 检查项数量
维度加权分 = 维度得分 × 权重
综合评分 = Σ(各维度加权分)
```
