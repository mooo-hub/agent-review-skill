# 子Agent审查检查清单

> **用途**：审查子Agent是否符合Specialist角色规范
> **版本**：v1.0 | 2026-06-17
> **来源**：[[../knowledge/四角色架构规范]]

---

## 核心审查原则

```
┌──────────────────────────────────────────────────────────────┐
│                    审查核心原则                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【原则一：单一职责】                                          │
│  每个子Agent只做一件事，能否用一个动词描述其全部工作？          │
│                                                              │
│  【原则二：契约优先】                                          │
│  输入/输出schema先于实现定义，Agent之间通过契约解耦            │
│                                                              │
│  【原则三：禁止越权】                                          │
│  不得直接响应用户、不得调用其他子Agent、不得修改任务范围        │
│                                                              │
│  【原则四：失败上报】                                          │
│  失败时上报错误，不自行决策重试策略                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度一：输入契约（权重25%）

```
┌──────────────────────────────────────────────────────────────┐
│                    输入契约检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [S-1.1] 定义了期望接收的输入数据结构（JSON schema）         │
│    ├─ 检查：是否有"输入契约"章节                              │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：使用JSON格式定义输入结构                      │
│                                                              │
│  □ [S-1.2] 标注了必填/选填字段                                 │
│    ├─ 检查：是否区分必填和选填字段                            │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：在字段描述中标注"(必填)"或"(选填)"            │
│                                                              │
│  □ [S-1.3] 定义了输入校验规则                                  │
│    ├─ 检查：是否说明如何校验输入数据                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：说明字段格式要求和校验方式                    │
│                                                              │
│  □ [S-1.4] 校验失败有明确错误返回（status=failed）             │
│    ├─ 检查：是否定义输入校验失败时的处理                      │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：返回status=failed并说明原因                   │
│                                                              │
│  □ [S-1.5] 输入来源明确（来自主Agent调度/其他Agent输出）        │
│    ├─ 检查：是否说明输入数据从哪里来                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：说明来自主Agent调度或上游Agent输出             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度二：输出契约（权重25%）

```
┌──────────────────────────────────────────────────────────────┐
│                    输出契约检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [S-2.1] 定义了固定的输出格式（JSON schema或模板）           │
│    ├─ 检查：是否有"输出契约"章节                              │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：使用JSON格式定义输出结构                      │
│                                                              │
│  □ [S-2.2] 输出包含task_id字段（与输入一致）                   │
│    ├─ 检查：输出JSON是否包含task_id                           │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：task_id与输入保持一致                         │
│                                                              │
│  □ [S-2.3] 输出包含status字段（success/failed/partial）        │
│    ├─ 检查：输出JSON是否包含status                            │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：定义三种状态及含义                            │
│                                                              │
│  □ [S-2.4] 输出包含error字段（失败时有错误详情）               │
│    ├─ 检查：输出JSON是否包含error字段                         │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：失败时填充error详情                           │
│                                                              │
│  □ [S-2.5] 使用ALWAYS标注输出格式约束                          │
│    ├─ 检查：是否有明确的格式约束说明                          │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：使用"ALWAYS输出xxx格式"表述                   │
│                                                              │
│  □ [S-2.6] 输出模板是具体内嵌的，非描述性的                    │
│    ├─ 检查：输出模板是否直接内嵌在SKILL.md中                  │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：直接内嵌JSON模板，而非文字描述                │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 维度三：职责边界（权重30%）

```
┌──────────────────────────────────────────────────────────────┐
│                    职责边界检查项                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  □ [S-3.1] 一句话能描述全部工作（<=2个动词）                   │
│    ├─ 检查：是否能在一句内描述子Agent的全部工作                │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：简洁描述如"执行性格画像分析"                  │
│                                                              │
│  □ [S-3.2] 有明确的NEVER约束（>=1条）                          │
│    ├─ 检查：是否定义了禁止事项                                │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：在"禁止任务"或"不负责"中列出                  │
│                                                              │
│  □ [S-3.3] 不直接响应用户（仅由主Agent调用）                   │
│    ├─ 检查：是否明确不直接响应用户                            │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：明确"仅由主Agent调用"                         │
│                                                              │
│  □ [S-3.4] 不调用其他子Agent（防止环路）                       │
│    ├─ 检查：是否明确不调用其他子Agent                         │
│    ├─ 错误级别：error                                         │
│    └─ 正确做法：在"禁止任务"中列出                            │
│                                                              │
│  □ [S-3.5] 不自行修改任务范围                                  │
│    ├─ 检查：是否明确不修改任务范围                            │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：严格按输入契约执行                            │
│                                                              │
│  □ [S-3.6] 失败时上报错误，不自行决策重试策略                  │
│    ├─ 检查：是否明确失败时上报而非自行重试                    │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：返回error让主Agent决策                        │
│                                                              │
│  □ [S-3.7] 不做策略判断（决策权归主Agent）                     │
│    ├─ 检查：是否明确不做策略判断                              │
│    ├─ 错误级别：warning                                        │
│    └─ 正确做法：只执行业务逻辑，策略由主Agent决定              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 常见问题与正确做法

| 问题 | 错误做法 | 正确做法 |
|------|----------|----------|
| 无NEVER约束 | 只写职责不写禁止 | 明确列出禁止事项 |
| 输出格式模糊 | "输出分析结果" | 输出具体JSON schema |
| 职责过多 | 一个Agent做三件事 | 拆分为多个子Agent |
| 无知识验证 | 直接开始执行 | 先读取知识文件并输出规则 |

## 评分标准

| 检查项数量 | 评分计算 |
|-----------|---------|
| pass数量 | 100分/项 |
| warning数量 | 60分/项 |
| error数量 | 0分/项 |

```
维度得分 = Σ(检查项得分) / 检查项数量
维度加权分 = 维度得分 × 权重
综合评分 = Σ(各维度加权分)
```
